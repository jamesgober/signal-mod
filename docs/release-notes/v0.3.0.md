# mod-signal v0.3.0 - Minimum Viable Implementation

**Date:** 2026-05-19
**Compare:** `v0.2.0...v0.3.0`

## Headline

The first release that contains actual code. The public surface
specified at `v0.2.0` is now implemented, tested, and documented.
`Cargo.toml` version bumped from `0.2.0` to `0.3.0`. Consumers
upgrading from `0.2.0` see source materialize in `src/`; consumers
upgrading from `0.1.0` should read the public-API section of
[`REPS.md`](../../REPS.md) before integrating.

The release closes the `0.3.0` milestone defined in
`.dev/ROADMAP.md`:

1. Implement the core API surface.
2. Unit tests covering all public methods.
3. Integration tests for the primary use case.
4. First-class documentation for every public item.

All four ship.

## What changed

### Added - core source under `src/`

Seven modules, each one entry-point per public type:

- `signal.rs` - `Signal`, `SignalSet`, `SignalSetIter`. Bit-packed
  set is const-constructible and `Copy`. Named constructors
  `empty()` / `all()` / `graceful()` / `standard()`. `IntoIterator`
  on `SignalSet` and `&SignalSet`. Per-variant Unix number table and
  Windows event mapping documented in rustdoc.
- `reason.rs` - `ShutdownReason` with `Display`,
  `description()`, `is_signal()`.
- `state.rs` (private) - the shared `Arc<Inner>` state machine.
  Atomic flag, mutex-protected reason + time, condvar for blocking
  waits. When `tokio` is enabled, `Inner` also owns a
  `tokio::sync::broadcast::Sender<ShutdownReason>` for async wait.
- `token.rs` - `ShutdownToken` (observer) and `ShutdownTrigger`
  (initiator) handles. Cloneable. Token has `is_initiated`,
  `reason`, `elapsed`, `wait_blocking`, `wait_blocking_timeout`, and
  a feature-gated async `wait`. Trigger has `trigger` (returns
  `bool`) and `is_initiated`.
- `hook.rs` - `ShutdownHook` trait, `FnHook` closure adapter,
  `hook_from_fn` convenience constructor.
- `coord.rs` - `Coordinator`, `CoordinatorBuilder`, `Statistics`.
  Owns the hook list (`Mutex<Vec<Box<dyn ShutdownHook>>>`),
  installed-flag (`AtomicBool`), hooks-completed counter
  (`AtomicUsize`). `run_hooks` sorts descending by priority
  (`sort_by_key` + `Reverse`) and honors the graceful budget.
- `error.rs` - `Error` enum (`AlreadyInstalled`,
  `SignalRegistration`, `InvalidState`, `Timeout`, `NoRuntime`),
  `Display + std::error::Error` impls, `Result<T>` alias.

### Added - `Coordinator::install` feature dispatch

The install path is compile-time selected:

| Active features                                  | Back-end                                          |
| ------------------------------------------------ | ------------------------------------------------- |
| `tokio` (default)                                | `tokio::signal::{unix,windows}` + `tokio::spawn`  |
| `async-std` (and not `tokio`)                    | `signal-hook-async-std` on Unix; `ctrlc` on Win   |
| `ctrlc-fallback` (and neither runtime)           | `ctrlc::try_set_handler` (`Signal::Interrupt`)    |
| none of the above                                | returns `Error::NoRuntime`                        |

`tokio` takes precedence over `async-std` when both are enabled,
matching the convention `proc-daemon` follows. Failed registrations
revert the installed flag so a subsequent `install` call can retry.

### Added - 30 tests

- 21 unit tests across `signal`, `reason`, `hook`, `coord` modules.
- 7 integration tests in `tests/coordinator_integration.rs`:
  token-clone sharing, trigger idempotence, hook reason observation,
  statistics tracking, signal-set iteration, `NoRuntime` error
  surface (cfg-gated on missing back-end features), blocking-wait
  timeout.
- 2 doctests: the crate root quick-start and `hook_from_fn`.

All pass on Windows + tokio, Windows + async-std, Windows +
ctrlc-fallback, and Windows with no back-end feature.

### Added - `examples/graceful_shutdown.rs`

The canonical pattern in 35 lines: build a coordinator with two
priority-ordered hooks, install signal handlers, `token.wait().await`,
then `run_hooks(reason)`.

### Added - `docs/API.md`

Offline API mirror covering every public type, the `install`
dispatch table, the feature matrix, and the dependency rationale.

### Changed - `lib.rs`

- Crate-level rustdoc expanded with a Quick Start doctest gated on
  the `tokio` feature (the same pattern `proc-daemon v1.1.2`
  adopted: `#[cfg(feature = "tokio")] async fn main` plus a
  `#[cfg(not(feature = "tokio"))] fn main() {}` fallback so the
  doctest stays green on no-feature builds).
- Crate-level attributes streamlined: `forbid(unsafe_code)`,
  `warn(missing_docs)`. The earlier `warn(clippy::pedantic)` was
  removed because `.dev/DIRECTIVES.md` only requires default-strict
  clippy, and pedantic surfaces stylistic noise (`must_use_candidate`,
  `unnecessary_sort_by`, `iter_without_into_iter`, `unused_self`)
  that costs more than it catches at this stage.

### Changed - `Cargo.toml`

- `parking_lot 0.12` is now a mandatory runtime dependency. Pulled
  in for `Mutex` and `Condvar` (used by `Inner` and by the blocking
  wait path).
- Optional deps: `tokio 1.40` (`sync`, `signal`, `rt`, `time`,
  `macros`), `async-std 1.12`, `futures 0.3`, `ctrlc 3.4`.
- Optional target-gated unix deps: `signal-hook 0.3`,
  `signal-hook-async-std 0.2`.
- Features: `std` (default), `tokio` (default), `async-std`,
  `ctrlc-fallback`. The `async-std` feature additionally pulls in
  `ctrlc` because async-std lacks a native Windows signal stream
  and the Windows path falls back to a synchronous `ctrlc` handler.
- `[package.metadata.docs.rs]` now lists explicit features
  (`std`, `tokio`, `ctrlc-fallback`) and pins
  `default-target = "x86_64-unknown-linux-gnu"`. `async-std` is
  excluded for the same reason `proc-daemon v1.1.1` excluded it -
  its transitive `rustix 0.37.x` uses internal `rustc_*` attributes
  that current nightly rejects.

### Internal

- All `unsafe` is delegated to `tokio`, `signal-hook`,
  `signal-hook-async-std`, and `ctrlc`. `mod-signal` itself remains
  `#![forbid(unsafe_code)]`.
- `Coordinator::install` uses `compare_exchange` to guard against
  concurrent double-install and reverts the flag on back-end
  failure so retry is possible.
- Hook sort is `sort_by_key(|h| Reverse(h.priority()))` per
  clippy's `unnecessary_sort_by` recommendation; stable within a
  priority by `Vec::sort_by_key`'s stability guarantee.

## Migration from `0.2.0`

`0.2.0` was design-only. There is nothing to migrate from. Add the
dep:

```toml
[dependencies]
mod-signal = "0.3"
```

Default features (`std + tokio`) cover the most common path. To
swap runtimes:

```toml
mod-signal = { version = "0.3", default-features = false, features = ["std", "async-std"] }
```

For non-async code:

```toml
mod-signal = { version = "0.3", default-features = false, features = ["std", "ctrlc-fallback"] }
```

## Stability commitment

`0.3.0` is a minor release in the 0.x line. The pre-1.0 stability
convention for this crate is:

- **Minor (`0.x.0`)** - public surface may add and rearrange.
- **Patch (`0.x.y`)** - bug fixes only; no surface change.

API freeze begins at `1.0.0-rc.1`.
